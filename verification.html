Verification Admin Commands

Commands for managing verified members, primarily for backup or transfer purposes.

Default Prefix: .

.transferverified

Syntax: .transferverified

Aliases: .tv, .syncmembers

Adds members who verified for a specific *old* server (source_guild_id) AND consented to auto-join, into the *current* server where the command is run. Intended as a one-time transfer or backup restoration tool.

Required Permissions:

  • Bot: Create Instant Invite or Manage Guild / Add Members (in the current/target server)
  • User: Administrator (in the current/target server)

Parameters:

  • (Required): The exact server ID of the *old* server where users originally verified with this bot.

Examples:

.transferverified 111222333444555666

Important Notes:
- Run this command in the **NEW** server you want to add members to.
- Only users who verified for the specific `source_guild_id` *and* clicked "Yes, Allow Auto-Join" on the consent page will be added.
- This command can only be successfully run **once per source_guild_id** for this target server to prevent accidental mass adds.
- Requires the bot to have appropriate permissions in the **target (current)** server.
- May take a long time and hit Discord rate limits if many users need adding.

.addverified

Syntax: .addverified

Aliases: None

(Bot Owner Only) Attempts to add ALL users who have ever verified with this bot instance (on any server) AND consented to auto-join, into the specified target server.

Required Permissions:

  • Bot: Create Instant Invite or Manage Guild / Add Members (in the target server)
  • User: Bot Owner

Parameters:

  • (Required): The ID of the server to add users to.

Examples:

.addverified 999888777666555444

WARNING: This is a powerful global command. Use with extreme caution as it attempts to add potentially many users. Primarily for specific bot administration purposes.
