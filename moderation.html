Moderation Commands

Tools to manage users, messages, and channels to maintain a healthy community.

Default Prefix: .

.kick

Syntax: .kick [reason]

Removes a specified member from the server.

Required Permissions:

  • Bot: Kick Members
  • User: Kick Members

Parameters:

  • (Required): The user to kick (@Mention, ID, Username#Tag).
  • [reason] (Optional): Why the user is being kicked. Logged and potentially DMed.

Examples:

.kick @User123 Spamming invites
.kick 987654321098765432 Repeated rule breaking

.ban

Syntax: .ban [days_to_delete] [reason]

Bans a member from the server, optionally deleting their recent messages.

Required Permissions:

  • Bot: Ban Members
  • User: Ban Members

Parameters:

  • (Required): The user to ban (@Mention, ID, Username#Tag). Can also be a User ID for someone not in the server.
  • [days_to_delete] (Optional): Number of days (0-7) of messages to delete from the banned user. Defaults to 0.
  • [reason] (Optional): Why the user is being banned.

Examples:

.ban @BadUser#5678 1 Extreme toxicity
.ban 112233445566778899 Raid account
.ban @AnotherUser 7

.unban

Syntax: .unban [reason]

Unbans a user, allowing them to rejoin the server if they have an invite.

Required Permissions:

  • Bot: Ban Members
  • User: Ban Members

Parameters:

  • (Required): The ID of the user to unban. You cannot use mentions here.
  • [reason] (Optional): Why the user is being unbanned.

Examples:

.unban 112233445566778899 Appeal accepted

Tip: You can usually find a User ID by enabling Developer Mode in Discord, then right-clicking the user and selecting "Copy User ID".

.timeout

Syntax: .timeout [reason]

Aliases: .mute

Temporarily prevents a member from interacting (sending messages, speaking in VC). Uses Discord's built-in Timeout feature.

Required Permissions:

  • Bot: Moderate Members
  • User: Moderate Members

Parameters:

  • (Required): The user to timeout (@Mention, ID, Username#Tag).
  • (Required): How long to timeout for (e.g., 10m, 1h, 3d, 1w). Max is usually 28 days.
  • [reason] (Optional): Why the user is being timed out.

Examples:

.timeout @NoisyUser 30m Spamming chat
.timeout 123456789012345678 2h Arguing after warning

.untimeout

Syntax: .untimeout [reason]

Aliases: .unmute

Removes an active timeout from a member.

Required Permissions:

  • Bot: Moderate Members
  • User: Moderate Members

Parameters:

  • (Required): The user to remove timeout from (@Mention, ID, Username#Tag).
  • [reason] (Optional): Why the timeout is being removed.

Examples:

.untimeout @NoisyUser Timeout served

.warn

Syntax: .warn

Issues a formal warning to a member, which is stored by the bot.

Required Permissions:

  • Bot: [Send Messages, Embed Links recommended]
  • User: Kick Members

Parameters:

  • (Required): The user to warn (@Mention, ID, Username#Tag).
  • (Required): The specific reason for the warning.

Examples:

.warn @ChattyUser Please keep discussions in the correct channel.

Note: Warnings are tracked by the bot. Use .warnings to view them.

.warnings

Syntax: .warnings

Aliases: .warns, .infractions

Displays the list of warnings recorded for a specific member.

Required Permissions:

  • Bot: [Send Messages, Embed Links recommended]
  • User: Kick Members

Parameters:

  • (Required): The user whose warnings you want to view (@Mention, ID, Username#Tag).

Examples:

.warnings @ChattyUser

.clearwarnings

Syntax: .clearwarnings

Aliases: None

Removes *all* warnings for a specified member after confirmation.

Required Permissions:

  • Bot: [Send Messages, Add Reactions]
  • User: Administrator

Parameters:

  • (Required): The user whose warnings to clear (@Mention, ID, Username#Tag).

Examples:

.clearwarnings @ChattyUser

.unwarn

Syntax: .unwarn

Aliases: None

Removes a specific warning by its ID (obtained from .warnings).

Required Permissions:

  • User: Manage Messages

Parameters:

  • (Required): The user the warning belongs to.
  • (Required): The exact MongoDB Object ID of the warning.

Examples:

.unwarn @User123 60a1b2c3d4e5f6a7b8c9d0e1

.purge

Syntax: .purge [member]

Aliases: .clear, .clean

Deletes a specified number of messages (1-100) from the current channel, optionally filtering by member.

Required Permissions:

  • Bot: Manage Messages, Read Message History
  • User: Manage Messages

Parameters:

  • (Required): How many messages to delete (1-100).
  • [member] (Optional): Filter to only delete messages from a specific @User or ID.

Examples:

.purge 50
.purge 25 @SpamBot#1234

Note: Bots typically cannot bulk delete messages older than 14 days.

.lock

Syntax: .lock [channel] [reason]

Aliases: None

Prevents the @everyone role from sending messages in the specified channel (or current channel).

Required Permissions:

  • Bot: Manage Channels
  • User: Manage Channels

Parameters:

  • [channel] (Optional): The #channel to lock. Defaults to the current channel.
  • [reason] (Optional): Why the channel is being locked.

Examples:

.lock
.lock #general Spam cleanup

.unlock

Syntax: .unlock [channel] [reason]

Aliases: None

Restores permission for the @everyone role to send messages in the specified channel (or current channel).

Required Permissions:

  • Bot: Manage Channels
  • User: Manage Channels

Parameters:

  • [channel] (Optional): The #channel to unlock. Defaults to the current channel.
  • [reason] (Optional): Why the channel is being unlocked.

Examples:

.unlock
.unlock #general Cleanup complete
