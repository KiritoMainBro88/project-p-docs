Stream Monitor Commands

Configure notifications for when YouTube or TikTok creators go live.

Default Prefix: . (Main group command is notify)

{/* Adjusted ID to match convention and likely search index entry */}

.notify add

Syntax: .notify add [#channel]

Adds a creator to the notification list for a channel. The bot will periodically check if they are live and post a message.

Required Permissions:

  • User: Manage Guild

Parameters:

  • (Required): The platform ('youtube' or 'tiktok'). Case-insensitive.
  • (Required): The creator's identifier. Can be:
    • YouTube: Channel name (`PewDiePie`), handle (`@MrBeast`), channel ID (`UC...`), full channel URL (`https://youtube.com/c/...`).
    • TikTok: Username (`charlidamelio`), full profile URL (`https://www.tiktok.com/@...`).
  • [#channel] (Optional): The text channel where notifications should be sent. If omitted, uses the channel where the command was run.

Examples:

.notify add youtube MrBeast #streams
.notify add tiktok charlidamelio
.notify add youtube https://www.youtube.com/channel/UCX6OQ3DkcsbYNE6H8uQQuVA #general

Note: TikTok detection may be less reliable than YouTube due to frequent platform changes.

.notify remove

Syntax: .notify remove [#channel]

Aliases: .notify delete

Removes a creator from the notification list for a specific channel.

Required Permissions:

  • User: Manage Guild

Parameters:

  • (Required): The platform.
  • (Required): The identifier you need to remove. Try using the same name/URL/ID you used to add them, or check .notify list for the stored ID.
  • [#channel] (Optional): The channel the notification was configured for. Defaults to the current channel.

Examples:

.notify remove youtube MrBeast #streams

.notify list

Syntax: .notify list [#channel]

Aliases: None

Lists all creators being monitored for notifications in a specific channel (or the current channel if none specified).

Required Permissions:

  • User: None (Anyone can list)

Parameters:

  • [#channel] (Optional): The channel whose notification list you want to see. Defaults to the current channel.

Examples:

.notify list
.notify list #live-streams

.notify check

Syntax: .notify check [#channel]

Aliases: .notify status

Manually performs an immediate check for the live status of a creator that is currently being monitored.

Required Permissions:

  • User: Manage Guild

Parameters:

  • (Required): The platform.
  • (Required): The identifier of the monitored creator.
  • [#channel] (Optional): The channel the notification is configured for (needed to find the specific entry). Defaults to the current channel.

Examples:

.notify check youtube MrBeast #streams

This helps debug if you think a notification was missed, but doesn't guarantee a notification if they aren't currently live.
