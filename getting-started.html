Getting Started with Project P

1. Inviting the Bot

Adding Project P to your server is simple:

  1. Click the Invite Link: Use this link to start the process:
    Invite Project P to Your Server
  2. Select Your Server: Choose the Discord server you want to add the bot to from the dropdown menu. You must have "Manage Server" permissions on that server.
  3. Authorize Permissions: Review the permissions Project P requests. These are necessary for its commands to function correctly (e.g., `Kick Members` for `.kick`, `Manage Roles` for `.reactionrole`, `Manage Messages` for `.purge`). Click "Authorize".
  4. Confirmation: You might need to complete a CAPTCHA. Once done, the bot should appear in your server's member list!

2. Understanding the Prefix

Commands for Project P start with a prefix. The default prefix is a dot: .

To use a command, type the prefix followed immediately by the command name. For example: .help, .ping, .kick @User. You can change the prefix using .config prefix if you have `Manage Guild` permissions.

3. Verification (Using OAuth2)

Why Verify? Verification is used to securely link your Discord account with Project P. This allows the bot to confirm you are a real user and potentially enables advanced features in the future or helps secure the server against bots (depending on admin configuration). It may also be required to access certain channels or gain a specific role.

How to Complete Verification:

  1. Go to your Direct Messages (DMs) with the **Project P** bot. If you don't have one, try sending any message to the bot in a server it's in first, although the verify command *only works in DMs*.
  2. Type the command .verify [Server ID].
    • If the bot is only configured for verification on one server you both share, you might just need .verify.
    • If verification is set up on multiple servers, the bot will likely list the Server IDs, and you need to specify which one you want to verify for, e.g., .verify 123456789012345678.
  3. The bot will send you a reply in DMs containing instructions and a unique verification link button labeled "Verify Account". Click this button/link.
  4. You will be redirected to the official Discord website to authorize Project P. Review the requested permissions (usually 'identify' and 'guilds.join') and click "Authorize".
  5. You'll then see a confirmation page (it might ask about allowing the bot to auto-join servers for you - choose your preference).
  6. You can now close that web page and return to Discord.
  7. If the server administrator configured a role for verified users, the bot should automatically grant you that role shortly, giving you access.

Troubleshooting Verification:

  • Not receiving DMs? Go to User Settings -> Privacy & Safety -> Server Privacy Defaults -> Ensure "Allow direct messages from server members" is ON. You might need to temporarily enable this for the server where you first encountered the bot.
  • .verify doesn't work in server? The .verify command *must* be used in your DMs with the bot, not in a server channel.
  • Link Expired/Invalid? Authorization links are temporary. Simply run the .verify [Server ID] command in DMs again to get a fresh link.
  • Still Stuck? Contact the administrators of the specific Discord server you are trying to get verified in or ask in the bot's official Support Server.

4. Basic Usage

Once invited (and verified, if needed), you can start using commands in the server. Try these basics:

  • .help - Shows the main categories of commands.
  • .ping - Checks the bot's response time.
