Role Management Commands

Commands for managing roles, including setting up reaction roles.

Default Prefix: .

.reactionrole

Syntax: .reactionrole [subcommand] [options...]

Sets up or manages messages where users can react with an emoji to gain or lose a specific role.

Required Permissions:

  • Bot: Manage Roles, Add Reactions, Read Message History, [Maybe Send Messages/Embed Links depending on setup]
  • User: Manage Roles (or potentially Manage Server for setup)

Subcommands & Parameters (Example Structure - ADAPT TO YOUR BOT):

  • .reactionrole add : Adds a new reaction role mapping to an existing message.
  • .reactionrole remove : Removes a reaction role mapping.
  • .reactionrole create [channel] "Your message text here": Creates a new message specifically for reaction roles (bot might then prompt for emoji/role pairs).
  • .reactionrole list [channel] [message_id]: Lists active reaction roles on a message or channel.

Examples (ADAPT TO YOUR BOT'S SYNTAX):

.reactionrole add #roles 112233445566778899 ✅ @VerifiedRole
.reactionrole add #get-roles 998877665544332211 <:custom_emoji:123> @TeamBlue
.reactionrole create #assign-roles "React here for notification roles!"
.reactionrole remove #roles 112233445566778899 ✅

This command can be complex. Ensure you provide the correct Channel ID, Message ID, Emoji (including custom ones like `<:name:id>`), and Role (@Mention or ID).
